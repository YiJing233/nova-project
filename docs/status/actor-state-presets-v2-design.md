# Denova 内置状态系统预设 V2.2：面板、状态与用户布局

> 状态：已实施（2026-07-19）
>
> 覆盖预设：默认、修仙、西幻、末世、无限流
>
> 影响范围：内置 Actor State 预设、TRPG 嵌套数值读取、状态面板布局偏好；不改变 Actor 状态更新与回放协议

## 1. 设计目标

状态系统只维护“本回合结束后仍成立，并会影响后续行动”的事实。

本版遵守以下原则：

1. 同一个事实只保留一个归属，相似且同步变化的内容尽量合并。
2. 角色状态明确区分“面板”和“状态”，但不把这套字段机械套给所有题材。
3. 面板只保存会参与检定的有效值；状态只保存动态资源、效果和冷却。
4. 灵根、特殊体质、血脉、身份、外貌、能力说明等仍使用文本或记录，不为了结构统一而数值化。
5. 技能、物品和关系归入所属角色；地点与势力归入世界实体；当前场景、世界局势和任务归入故事状态。
6. Schema 的 `group`、`display` 只提供默认分区和渲染提示，最终排序由用户在状态面板中决定。
7. `update_instruction` 只说明结构、量表或真正有用的更新约束，不重复通用套话。

状态预设不保存纪要、总结、完整经历、行动选项、幕后计划和回合流水。这些继续由 Turn、Director、Lore、choices 等组件负责。

## 2. 固定模板边界

五套预设都使用五种模板：

| Template ID | 名称 | 实例方式 | 内容 |
| --- | --- | --- | --- |
| `story_context` | 故事状态表 | 单例 | 当前场景、世界局势、当前任务、题材级状态 |
| `protagonist` | 主角状态表 | 单例 | 人物设定、面板、状态、技能、物品、关系 |
| `important_character` | 重要角色状态表 | 每角色一个 Actor | 角色资料、面板、状态、对主角关系及自身持有记录 |
| `opponent` | 敌对对象状态表 | 每对象一个 Actor | 威胁、面板、状态、能力、弱点及持有记录 |
| `world_entities` | 世界实体表 | 单例 | 地点和势力 |

默认只创建三个 Actor：

| Actor ID | Template ID |
| --- | --- |
| `protagonist` | `protagonist` |
| `story` | `story_context` |
| `world` | `world_entities` |

重要角色和敌对对象按需创建，不预建空 Actor。

## 3. 面板与状态

### 3.1 面板

“面板”是检定侧的有效属性集合，例如等级、作品采用的属性、攻击 AC、防御 DC。它可以被等级、装备和当前状态修正，因此每个数值项至少表达：

```json
{
  "力量": {
    "基础值": 12,
    "当前值": 14,
    "修正说明": "巨力腰带 +2",
    "量表说明": "1–5 极弱；6–9 偏弱；10–11 常人；12–15 优秀；16–19 卓越；20+ 超凡"
  }
}
```

- `基础值`：不计算临时修正时的值。
- `当前值`：当前检定实际读取的结算后有效值。
- `修正说明`：解释装备、效果或其他来源，避免出现不可追溯的数值。
- `量表说明`：给出该数值的区间语义；默认 TRPG 的等级、六维和 AC/DC 均随预设提供。
- 题材没有数值规则时，面板允许为空。
- 只有作品明确量化的项目才进入面板，不能因为存在 object 就自行补齐六维。

### 3.2 状态

“状态”是动态变化集合，固定使用三个分区：

```json
{
  "资源": {
    "生命": { "当前值": 18, "上限": 30 },
    "法力": { "当前值": 6, "上限": 10 }
  },
  "效果": {
    "poison_001": {
      "名称": "中毒",
      "类型": "异常",
      "剩余": "3轮",
      "层数": 2,
      "影响": "体质检定 -2"
    }
  },
  "冷却": {
    "ability_fireball": {
      "名称": "火球术",
      "剩余": 2,
      "单位": "轮"
    }
  }
}
```

- `资源`：生命、法力、境界进度、法术位、积分等作品实际采用的动态量。
- `效果`：Buff、Debuff、伤势、异常、祝福、诅咒、污染、封印、死亡标记等。
- `冷却`：键优先复用稳定 ability ID 或 item ID。
- 技能说明仍在“技能与能力”，物品数量与耐久仍在“重要物品”；状态只保存其即时效果或冷却。
- 原先的文本“当前状态”被该 object 取代；“当前处境”继续作为叙事字段，承接危险、限制与直接压力。

## 4. 默认预设字段

### 4.1 主角状态表：10 个字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| 姓名 | string | 主角姓名 |
| 基本身份 | string | 年龄、性别、职业、种族、社会身份 |
| 外貌与装扮 | string | 固有外貌、衣着、装备外观、伪装 |
| 性格与背景 | string | 稳定性格、关键背景 |
| 面板 | object | 检定有效值 |
| 状态 | object | 资源、效果、冷却 |
| 当前处境 | string | 优势、危险、限制、直接压力 |
| 技能与能力 | object | 主角持有的能力记录 |
| 重要物品 | object | 主角持有的重要物品 |
| 关系 | object | 主角对其他 Actor 或势力的单向关系 |

默认 TRPG 面板提供等级、力量、敏捷、体质、智力、感知、魅力、攻击 AC、防御 DC；这只属于默认 TRPG 预设，不是所有题材的公共六维。

### 4.2 故事状态表：9 个字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| 当前时间 | string | 日期、时段或阶段，只保留有行动意义的精度 |
| 当前详细地点 | string | 具体地点及必要的上级范围 |
| 当前事件 | string | 当前事件、直接压力、下一步问题 |
| 在场角色 | list | 当前可感知或互动的 Actor ID |
| 当前场景要素 | object | 会影响下一步行动的现场实体 |
| 当前生效规则 | list | 当前场景或世界中实际约束行动的规则 |
| 世界背景 | string | 时代、文明、力量体系及大范围环境 |
| 世界局势 | string | 跨场景威胁、环境变化和倒计时 |
| 当前任务 | object | 已成立且仍需推进的任务 |

不拆分当前大区、当前区域、当前房间、天气、场景压力等低价值字段。需要时合并进入当前地点、当前事件或世界局势。

### 4.3 重要角色状态表：13 个字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| 基本身份 | string | 年龄、职业、种族、组织及公开身份 |
| 外貌与装扮 | string | 辨识特征、装束、装备外观、伪装 |
| 性格与背景 | string | 稳定性格与关键来历 |
| 出场与位置 | string | 在场、离场、失联、死亡及位置 |
| 面板 | object | 该角色的检定有效值 |
| 状态 | object | 该角色的资源、效果、冷却 |
| 当前目标与处境 | string | 已确认目标、压力、优势与限制 |
| 与主角关系 | string | 关系类型、态度、阶段、承诺、边界、矛盾 |
| 对主角好感度 | number | 0..100 固定量表 |
| 对主角的已知信息 | list | 确实得知且会影响判断的信息，最多 6 项 |
| 技能与能力 | object | 该角色自己的能力 |
| 重要物品 | object | 该角色持有的重要物品 |
| 关系 | object | 该角色对主角以外目标的单向关系 |

不保留“对主角的怀疑”或“误解”专用字段。已经影响行动的内容进入“与主角关系”或“当前目标与处境”。

### 4.4 敌对对象状态表：10 个字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| 类型与外观 | string | 类型、身份、辨识特征 |
| 存续与位置 | string | 活跃、撤退、被俘、失效、消灭及位置 |
| 威胁态势 | string | 威胁级别、警戒或追击状态、当前目标 |
| 面板 | object | 对抗检定有效值 |
| 状态 | object | 资源、伤势、控制、阶段、异常、冷却 |
| 行动模式 | string | 已观察或确认的行为规律 |
| 弱点与退场条件 | string | 弱点与击败、驱散、封印、谈判、摆脱条件 |
| 技能与能力 | object | 对象自身能力 |
| 重要物品 | object | 对象持有的重要物品 |
| 关系 | object | 对象对其他 Actor 或势力的关系 |

“威胁等级”和“威胁状态”合并为“威胁态势”，使用低、中、高、致命四档并附带当前行为描述。

### 4.5 世界实体表：2 个字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| 地点记录 | object | 地点名称、类型、所属范围、状态、风险、特征与通路 |
| 势力记录 | object | 势力名称、类型、特征、存续、立场、范围与行动 |

地点和势力处于同一个 world Actor，但保留不同记录结构。

## 5. 数值量表

“对主角好感度”使用固定 0..100 量表：

| 区间 | 语义 |
| ---: | --- |
| 0–20 | 敌视或强烈排斥 |
| 21–40 | 戒备或疏远 |
| 41–60 | 中立、熟悉或可合作 |
| 61–80 | 友好且愿意信任 |
| 81–90 | 亲近并会主动支持 |
| 91–100 | 深厚羁绊；除非发生根本冲突通常不会动摇 |

一般事件变化 1–5，重大选择变化 6–15；普通同场、寒暄或没有关系结果的互动不自动增加。好感度不等同于信任、服从或恋爱关系。

面板和状态 object 中的作品内数值服从作品自己的量纲，不宣称跨作品统一语义。

## 6. 题材适配

| 预设 | 面板默认 | 状态默认 | 额外文本或对象 |
| --- | --- | --- | --- |
| 默认 TRPG | 等级、六维、攻击 AC、防御 DC | 生命、法力、效果、冷却 | 无 |
| 修仙 | 境界显示；不提供六维 | 境界进度、效果、冷却 | 灵根与特殊体质；角色根基/传承；本相；修行世界状态 |
| 西幻 | 职业等级、攻击 AC、防御 DC；其他属性按设定增加 | 生命、效果、冷却；法力/法术位按设定增加 | 超凡来源、血脉契约、阵营职责、生物特性、世界秩序 |
| 末世 | 空面板；只接受规则明确量化的生存/战斗项 | 空资源骨架、效果、冷却 | 生存专长与职责、变异特征、灾变局势、基地状态 |
| 无限流 | 空面板；只接受空间明确公布的评级或属性 | 空资源骨架、效果、冷却 | 空间身份、队伍角色、规则实体、空间状态、当前副本 |

修仙不会把身法、体魄、根骨、悟性、神识、气运默认量化。末世不会为饥饿、口渴、疲劳各建一个数值。无限流不会自行生成六维或战力评分。

字段规模：

| 预设 | 主角 | 故事 | 重要角色 | 敌对对象 | 世界实体 |
| --- | ---: | ---: | ---: | ---: | ---: |
| 默认 | 10 | 9 | 13 | 10 | 2 |
| 修仙 | 11 | 10 | 14 | 11 | 2 |
| 西幻 | 11 | 10 | 14 | 11 | 2 |
| 末世 | 10 | 11 | 14 | 11 | 2 |
| 无限流 | 11 | 11 | 14 | 11 | 2 |

所有内置模板均不超过 14 个顶层字段，不使用 `hidden` 字段。

## 7. TRPG 嵌套数值绑定

面板是 object，TRPG modifier 和公式项通过可选 `value_path` 读取内部数值：

```json
{
  "source": "actor",
  "field_id": "面板",
  "value_path": ["力量", "当前值"],
  "effect": "advantage"
}
```

规则：

1. 顶层 number 字段继续只写 `field_id`。
2. 提供 `value_path` 时，`field_id` 必须指向 object。
3. modifier 和 `change_formula.terms` 支持嵌套读取；最终 outcome delta 的目标仍必须是顶层 number，避免隐式改写复杂 object。
4. 运行时优先读取 Actor 当前值，缺失时读取字段默认值；路径不存在或末端不是 number 时按 required 规则报错或给出警告。
5. 审计结果保留 `field_id` 与 `value_path`，数值来源可追踪。

## 8. 状态面板布局

### 8.1 Schema 与布局的边界

- `fields` 数组顺序只是首次展示的兜底顺序。
- `group` 保留为默认分区提示，`display` 保留为默认渲染器提示。
- 新预设不生成 `order` 或 `display_groups`。
- 旧 Beta 数据中的 `order`、`display_groups` 会被接受但忽略，规范化后不再输出。
- 预设结构编辑器不再提供字段排序或分组排序控件。

### 8.2 用户布局偏好

状态面板的“自定义布局”支持：

1. 拖动整个分区改变分区顺序。
2. 拖动字段改变组内顺序。
3. 把字段拖入其他已有分区。
4. 使用键盘拖拽，或在窄屏通过上移、下移、前一分区、后一分区按钮完成同样操作。
5. 一键恢复 Schema 兜底布局。

持久化键由“当前故事 ID + Template ID”组成：

- 同一故事中，使用相同模板的 Actor 共享布局。
- 不同故事互不影响。
- Schema 新增字段时，字段追加到其默认分区；删除字段时自动忽略旧记录。
- 布局只保存在本地 UI 偏好中，不进入 Story、Actor State、提示词或模型上下文。

## 9. 内嵌记录约定

技能、物品、关系、任务、地点和势力继续使用 object，键为稳定 ID。

| 记录 | 主要列 | 默认上限 |
| --- | --- | ---: |
| 技能与能力 | 名称、类型、掌握或当前状态、效果、代价与限制、来源 | 主角 12；其他 6 |
| 重要物品 | 名称、类型、数量或状态、作用、限制、来源 | 主角 16；重要角色 6；敌对对象 4 |
| 关系 | 类型、阶段、好感度、态度、边界或承诺、主要矛盾 | 主角 12；重要角色 6；敌对对象 4 |
| 当前任务 | 名称、类型、状态、目标与进度、时限、奖惩、关联 ID | 8 |
| 地点记录 | 名称、类型、所属范围、状态、风险、特征、通路 | 16 |
| 势力记录 | 名称、类型、特征、存续、立场、范围、行动 | 8 |

物品由存放它的 Actor 持有；转移时同轮从原持有者移除并写入新持有者。关系始终表示当前 Actor 对目标的单向视角，不自动镜像。

## 10. 兼容与实施边界

1. 保留五个内置预设 ID、五个模板 ID，以及 `story`、`protagonist`、`world` 三个基础 Actor ID。
2. 保留“当前详细地点”和“当前事件”字段名，继续满足运行时基础依赖。
3. 已开始故事继续使用冻结的故事级 Schema，不自动迁移为面板/状态结构。
4. 新结构只影响新故事及用户主动恢复的内置预设；现有同 ID 用户覆盖不自动替换。
5. `order`、`display_groups` 的忽略只影响展示，不改变任何状态值或回放结果。
6. 布局偏好当前保存在浏览器本地；清理站点数据或更换设备后会回到默认布局。
